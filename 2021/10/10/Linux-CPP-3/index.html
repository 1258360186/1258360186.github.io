第三讲: GCC编译器

前言：

  1. GCC 编译器支持编译 Go、Objective-C，Objective-C ++，Fortran，Ada，D 和 BRIG（HSAIL）
    等程序；
  2. Linux 开发C/C++ 一定要熟悉 GCC
  3. VSCode是通过调用GCC编译器来实现C/C++的编译工作的；

实际使用中：

  • 使用 gcc 指令编译 C 代码
  • 使用 g++指令编译 C++ 代码

3.1 编译过程

  1. 预处理-Pre-Processing //.i文件
  2. 编译-Compiling // .s文件
  3. 汇编-Assembling // .o文件
  4. 链接-Linking // bin文件

    3.2 g++重要编译参数

  5. -g 编译带调试信息的可执行文件
  6. -O[n] 优化源代码
  7. -l 和 -L 指定库文件 | 指定库文件路径
  8. -I 指定头文件搜索目录
  9. -Wall 打印警告信息
  10. -w 关闭警告信息
  11. -std=c++11 设置编译标准
  12. -o 指定输出文件名
  13. -D 定义宏示例代码：注：使用 man gcc 命令可以查看gcc英文使用手册，见下图
